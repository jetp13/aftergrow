# Afterglow 全域設計系統規範

## 設計理念

> 黑色是夜的安靜,粉霓是夜裡的方向。Glow 並非炫光,而是「呼吸的證據」。

- Glow = Flow
- 可讀性優先於裝飾
- 平衡夜間安全與柔亮美感
- 所有介面皆應具備「呼吸節奏」的動態節律

## 色彩系統

### 主色與副色
- **主色 Glow Pink**: `#FFB7C5` - Glow 主光源、重點字
- **副色 Rose Gold**: `#EFA7A7` - 補光、分隔線、框線
- **背景 Deep Black**: `#0B0B0D` - 全域背景
- **文字主色 Mist White**: `#FAFAFA` - 主內文字體
- **次文字色 Soft Gray**: `#A0A0A0` - 副標題與描述
- **高亮色 Neon Glow**: `#FFC8E1` - Hover／光暈外圈

### 應用原則
1. 光暈不應遮蔽文字
2. 對比維持 5:1
3. 可感知，不可刺眼
4. Glow 為一種節奏語言，而非視覺特效

## 字體系統

| 層級 | 字體 | 權重 | 顏色 | 行距 | 用途 |
|------|------|------|------|------|------|
| H1 | F1 幾何極簡體 | 700 | #FAFAFA | 1.3 | 主標題 |
| H2 | F2 細體灰字 | 600 | #A0A0A0 | 1.4 | 副標題 |
| Body | F2 細體 | 400 | #FAFAFA | 1.6 | 內文 |
| Caption | F3 Sans / Mono | 400 | #A0A0A0 | 1.2 | 註解／標籤 |

## 核心元件

### 🔘 Glow Capsule Button
- 層級: L2 + L3
- 霓虹膠囊按鈕
- 用途: 導覽與互動核心

### 🫧 Breathing Ball
- 層級: L4
- 呼吸節奏動畫球
- 用途: 冥想主介面

### 🧠 Module Frame
- 層級: L1 + L2
- 模組框線 Glow 邊
- 用途: 區塊分層與結構輔助

## 開發規範

### 技術要求
- 程式碼以原生 HTML、CSS、JavaScript 撰寫
- 不得引入框架
- 採用多頁面應用程式（MPA）架構
- 頁面跳轉使用 `window.location.href`

### 設計原則
- 嚴格遵照 UI 設計稿
- 不得自行修改顏色、大小、字體、排版或互動邏輯
- 不要替換或優化任何設計細節
- 不要添加任何額外元素或動畫
- 所有標題、副標題、字體設計必須以 AfterGrow CI 與標準字為基準

### 中文排版
- 加強主副標題的階層關係與字體比例
- 確保排版清晰、避免突兀
